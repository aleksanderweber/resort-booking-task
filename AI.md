# AI Usage

I used ChatGPT as a supporting tool while working on this task.

## How AI was used

AI was used mainly for:
- discussing implementation options,
- refining the project structure,
- reviewing trade-offs between simpler and more abstract solutions,
- suggesting improvements to API design and component structure,
- helping shape test coverage,
- reviewing README wording and documentation structure.

## What I kept under my control

All final implementation decisions were made by me.

I used AI as a thinking and drafting assistant, not as a blind code generator. In particular, I reviewed and adjusted suggested solutions to keep the project aligned with the task requirements:
- keep the solution simple,
- avoid unnecessary abstraction,
- keep the architecture right-sized,
- focus on core functionality and automated tests.

I also verified the behavior by running the application and tests locally, and adjusted the implementation when needed.

## Examples of AI-assisted topics

Examples of topics I used AI for:
- how to structure a minimal Angular + Express solution,
- how to keep the backend simple while still separating file loading, map parsing, and booking logic,
- how to implement a single startup command from the project root,
- how to organize backend and frontend tests,
- how to phrase design decisions and project notes clearly.

## Development approach

My goal when using AI was to speed up routine work and challenge design choices, while still keeping the solution understandable and maintainable.

Where an AI suggestion felt too complex for the scope of the task, I simplified it before using it.
